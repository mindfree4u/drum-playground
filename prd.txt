# 드럼놀이터 예약시스템 PRD (Product Requirements Document)

## Problem (문제점)
1. 기존 예약 시스템의 한계
   - 전화나 직접 방문을 통한 예약으로 인한 관리자의 업무 부담
   - 실시간 예약 현황 파악의 어려움
   - 예약 시간 충돌 가능성
   - 예약 정보 관리의 비효율성

2. 사용자 측면의 불편함
   - 예약 가능 시간 확인을 위한 반복적인 문의 필요
   - 예약 변경/취소 시 즉각적인 확인 어려움
   - 예약 현황 실시간 확인 불가

## Requirements (요구사항)

### 1. 사용자 관리
- 회원/비회원 구분
- 관리자 권한 설정
- 사용자 역할 구분 (일반 사용자, 관리자)

### 2. 예약 기능
- 실시간 예약 가능 시간 확인
- 레슨/연습 구분 예약
- Room A는 레슨 전용으로 제한
- 지나간 시간 예약 방지
- 예약 취소 기능
- 예약 변경 기능

### 3. 시간 관리
- 운영 시간 설정 (10:00 ~ 20:00)
- 예약 가능 시간대 설정
- 레슨 3시간 이내 취소 제한
- 일일 예약 횟수 제한 (레슨 1회, 연습 1회)

### 4. 방 관리
- Room A, B, C, E 구분
- 각 방별 예약 현황 관리
- 방별 예약 가능 유형 설정

### 5. UI/UX
- 직관적인 예약 테이블
- 모바일 대응 반응형 디자인
- 날짜 선택 및 시간대 선택 편의성
- 예약 상태 시각적 표시

## Goals (목표)

### 1. 운영 효율화
- 관리자의 예약 관리 업무 경감
- 실시간 예약 현황 파악 용이
- 예약 관련 문의 감소
- 예약 데이터의 체계적 관리

### 2. 사용자 경험 개선
- 24시간 예약 가능
- 실시간 예약 현황 확인
- 간편한 예약/취소/변경
- 모바일 환경 지원

### 3. 시스템 안정성
- 예약 충돌 방지
- 데이터 정확성 보장
- 시스템 안정적 운영
- 보안성 확보

### 4. 비즈니스 목표
- 예약률 향상
- 고객 만족도 증가
- 운영 효율성 증대
- 수익성 개선

## 기술 스택
- Frontend: React.js
- Backend: Firebase
- Database: Firestore
- Authentication: Firebase Auth
- Hosting: Firebase Hosting
